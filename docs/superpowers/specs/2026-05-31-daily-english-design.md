# Daily English Design

## Context

The app is inspired by the public learning workflow observed on https://dailydictation.com/: a home page with exercise categories, topic pages with level filters, lesson pages with dictation practice and transcript tabs, plus a lightweight ranking/progress view. This implementation must be an original TypeScript web app. It must not copy DailyDictation branding, proprietary data, audio files, images, user lists, source code, or exact visual design.

## Product Scope

Daily English is a static single page application for English dictation practice. Learners can browse topics, filter lessons by CEFR level, open a lesson, listen to sentence audio generated by the browser speech engine, type what they hear, check answers, reveal hints, move between sentences, view the full transcript, switch light/dark mode, and see locally saved progress.

## Architecture

The app uses Vite, React, TypeScript, React Router, Vitest, Testing Library, and GitHub Pages. Lesson content is local typed data in `src/data/lessons.ts`. Core answer checking is isolated in `src/lib/dictation.ts` so it can be tested without UI. Progress persistence is isolated in `src/lib/progress.ts` and uses localStorage through a small adapter. React pages compose shared components and keep UI state local.

## Pages

- Home: overview, learning steps, topic cards, featured lessons, and FAQ.
- Exercises: all topics and lessons with search and level filtering.
- Topic: topic detail and filtered lesson list.
- Lesson: dictation player, typed answer area, feedback, hints, navigation, transcript tab, and progress summary.
- Top Users: sample local leaderboard that explains real accounts are not connected.
- Login/Register: placeholder local account pages for visual completeness without backend auth.
- Not Found: helpful fallback.

## Data And Content

All sample lesson text is newly written for this project. Audio playback uses `SpeechSynthesisUtterance` in supported browsers, avoiding third-party audio assets. If speech synthesis is unavailable, the UI shows a clear message and still allows transcript reading.

## Styling

The design should feel close in information architecture to a dictation learning site, but use original colors, spacing, cards, iconography, and wording. It should support responsive desktop/mobile layouts, light/dark themes, keyboard-accessible controls, and stable component dimensions.

## Testing

Use TDD for core behavior:

- answer normalization ignores punctuation, capitalization, and extra spaces;
- answer scoring returns correct/missing/extra tokens and percentage;
- lesson progress calculates completion and accuracy;
- React smoke tests cover home navigation and lesson answer feedback.

## Deployment

The repository will be initialized locally and pushed to GitHub under `anhtnt90dev/daily-english` when authentication allows it. GitHub Pages deployment will use a GitHub Actions workflow that builds the Vite app and publishes `dist`.
